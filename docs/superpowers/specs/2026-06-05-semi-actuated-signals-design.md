# Semi-Actuated Traffic Signals — Design

**Date:** 2026-06-05
**Status:** Approved (brainstorming phase)

## Goal

Make auto-generated traffic signals behave like real isolated intersections:
the **major street rests in green**, and **side-street / minor phases are served
only on demand**, for as long as vehicles keep arriving (up to a cap). Eliminate
the unrealistic symptom of an empty side street getting a full fixed green every
cycle while the arterial waits at a red with no cross traffic present.

This is **semi-actuated** control (the common arterial case): the major phase is
non-actuated and rests; the minor phases are detector-actuated with
minimum-green, passage (gap) extension, and maximum-green.

## Background — what exists today

- Signals are auto-generated by `DefaultSignalConfig` (`internal/sim/signal.go`):
  approaches are bucketed by arrival-heading axis (N+S together, E+W together),
  one **fixed** 30 s green + 3 s yellow phase per axis, cycling forever.
- `SignalState.Advance(dt)` (signal.go:102) is a pure time-driven state machine:
  it has **no knowledge of vehicle positions**. It is called in tick step 0b
  (`World.Step`, world.go:670), *before* vehicles are bucketed by edge (step 2,
  world.go:714).
- Vehicles read the signal only through `SignalState.GreenFor(pos)` — actuation
  changes *when* phases switch, not how a vehicle reads them, so the
  vehicle-facing contract is unchanged.
- `SignalPhase` trace events are emitted on every phase/yellow transition
  (world.go:684–696). Variable (actuated) timing therefore replays correctly in
  `tracereplay` with **no trace-format change** — the events simply land at
  different sim-times.
- `network.Edge` carries `Class RoadClass` and `SpeedLimit` (network/types.go),
  both available to `DefaultSignalConfig` via `net`. This is enough to pick the
  major axis without threading OSM tags into the sim package.

## Major decisions (from brainstorming)

- **Default-on.** `DefaultSignalConfig` now produces a **semi-actuated** plan for
  every multi-phase signal. Single-phase (permanent-green) signals stay fixed —
  there is nothing to actuate.
- **Semi-actuated flavor.** One major (rest) phase + N actuated minor phases.
- **YAML overrides stay fixed-time.** A `--signals` override produces a
  `PlanFixed` config exactly as today (a mapper writing an override wants
  determinate timing). The override schema is unchanged. Opting an override into
  actuation is out of scope.
- **Determinism preserved.** Detection is a pure function of current vehicle
  positions, so `TestWorld_TraceDeterminism` still holds.
- All tuning values are named constants.

## Model

### Plan kind & config

`SignalConfig` (signal.go) gains:

```go
type PlanKind uint8
const (
    PlanFixed       PlanKind = iota // time-driven cycle (today's behavior)
    PlanSemiActuated                // major rests; minor phases actuated
)

type SignalConfig struct {
    // ...existing: IntersectionID, Phases, InitialMode...
    Plan          PlanKind // NEW; default PlanFixed
    MajorPhase    int      // NEW; index into Phases that rests in green
    MinGreen      float64  // NEW; minor-phase minimum green (s)
    MajorMinGreen float64  // NEW; major-phase min green before it may yield (s)
    Passage       float64  // NEW; gap time — green ends this long after the
                           //      passage zone clears (s)
    MaxGreen      float64  // NEW; minor-phase max green / max-out (s)
}
```

`SignalPhase` is unchanged. `GreenDur` is **ignored** for actuated phases (min /
passage / max replace it); `YellowDur` is still the per-phase clearance interval.
A `SignalConfig` with `Plan == PlanFixed` behaves exactly as today.

### Detector model

A vehicle on an approach edge `e` is at distance `d = e.Length − v.S` from the
stop line (end of edge). Two presence zones, evaluated across **all lanes** of
all of a phase's green edges:

- **`callDistance` (~60 m):** a vehicle with `d ≤ callDistance` raises a *call*
  on that phase (registers demand — used to decide whether to serve a minor
  phase and which phase to serve next).
- **`passageDistance` (~25 m):** a vehicle with `d ≤ passageDistance` *occupies*
  the phase (holds its green — resets the passage timer). As a standing queue
  discharges, the last vehicle leaves this zone, the passage timer runs out, and
  the phase gaps out — mirroring a stop-line loop detector.

Detection is computed once per tick, before signals advance, as two `[]bool`
slices per intersection (`called[phaseIdx]`, `occupied[phaseIdx]`), each a pure
function of vehicle positions.

### State machine (ModeNormal, PlanSemiActuated)

`SignalState` gains one field: `passageGap float64` — seconds since the current
minor phase's passage zone was last occupied. Flash/Off modes are unchanged
(actuation only refines `ModeNormal`).

Per tick, with detector inputs `called[]`, `occupied[]`:

- **Yellow:** run `YellowDur`, then pick the next phase via
  `nextActuatedPhase` (below). Reset `Elapsed = 0`, `passageGap = 0`.
- **Green, major phase (`PhaseIdx == MajorPhase`):**
  - `Elapsed += dt`.
  - Hold if `Elapsed < MajorMinGreen`.
  - Else if any minor phase is `called` → go yellow (terminate the major green
    to serve the call).
  - Else **rest**: stay green, hold `Elapsed` at `MajorMinGreen` (no float
    drift, no max-out — the major street rests indefinitely with no conflicting
    call).
- **Green, minor phase:**
  - `Elapsed += dt`; `passageGap = occupied[PhaseIdx] ? 0 : passageGap + dt`.
  - Terminate (→ yellow) when **max-out** `Elapsed ≥ MaxGreen`, **or**
    **gap-out** `Elapsed ≥ MinGreen && passageGap ≥ Passage`.

`nextActuatedPhase(curr)`: scan phases in cyclic order starting at `curr+1`;
return the first **minor** phase (`idx != MajorPhase`) that is `called`; if none
is called, return `MajorPhase`. Result: the controller serves each calling minor
phase in turn, then returns to the major street and rests there until the next
call. Minor phases with no demand are skipped entirely (never served).

### `Advance` API

`Advance(dt)` is kept for `PlanFixed` and is **unchanged**. Actuated states use a
new method:

```go
func (s *SignalState) AdvanceActuated(dt float64, called, occupied []bool)
```

`World.Step` dispatches on `s.Config.Plan`. Both paths are pure functions of
their inputs, so determinism holds.

## Architecture & data flow

### `DefaultSignalConfig` (rewritten tail)

Phase construction (axis bucketing) is **unchanged**. After building `phases`:

1. If `len(phases) <= 1`: return `PlanFixed` (permanent green — nothing to
   actuate), as today.
2. Pick `MajorPhase` = the phase whose approaches include the highest-priority
   road, scored by `max(RoadClass.Priority())` over the phase's green edges,
   tie-broken by highest `SpeedLimit`, then lowest phase index. This is the
   arterial axis that rests in green.
3. Return a `PlanSemiActuated` config with `MajorPhase` set and the default
   actuation timings (below). Per-phase `YellowDur` stays 3 s; `GreenDur` is left
   as set but ignored by the actuated advance.

### `World` integration (`world.go`, tick step 0b)

Before the signal-advance loop, compute detector inputs for actuated signals:

- At world build, precompute `detectorEdges` — the set of incoming edges of every
  `PlanSemiActuated` signal — so the per-tick scan is bounded.
- Per tick, one pass over `w.Vehicles` (non-despawned) records, for each detector
  edge, the **minimum** `d = Length − S` seen. From that, derive `called`
  (`min_d ≤ callDistance`) and `occupied` (`min_d ≤ passageDistance`) per phase
  by OR-ing across the phase's green edges.

Then in the advance loop: `PlanSemiActuated` → `AdvanceActuated(dt, called,
occupied)`; otherwise `Advance(dt)` as today. Phase/mode change-detection and
trace emission (world.go:684–704) are unchanged — they already fire on whatever
transitions the advance produces.

This adds one O(vehicles) pass plus O(detector edges) per tick — negligible
against the existing IDM / lane-change / routing work, and only touching edges
that feed actuated signals.

### No change required

- Vehicle decision logic (`stopDistanceForRed`, `effectiveControl`, gap
  acceptance) — reads `GreenFor` / `Mode`, both unchanged.
- Trace format & `tracereplay` — `SignalPhase` events already capture variable
  timing.
- Snapshot / renderer — reads `GreenFor` per approach; the HUD shows whatever the
  current phase is.
- Flash / Off modes and the `--signals` YAML override path (stays `PlanFixed`).

## Tuning defaults (named constants, `internal/sim`)

| Constant            | Value  | Meaning                                            |
|---------------------|--------|----------------------------------------------------|
| `actMajorMinGreen`  | 15 s   | major street holds ≥ this before yielding to a call |
| `actMinGreen`       | 7 s    | minor phase holds ≥ this once served                |
| `actPassage`        | 3 s    | green ends this long after the passage zone clears  |
| `actMaxGreen`       | 40 s   | minor-phase ceiling (max-out) under steady demand   |
| `actCallDistance`   | 60 m   | a vehicle within this raises a phase call           |
| `actPassageDistance`| 25 m   | a vehicle within this holds (extends) the green     |

Yellow stays 3 s (existing per-phase `YellowDur`). No separate all-red interval
is introduced (the yellow provides clearance) — noted as deferred.

Rationale: `actMajorMinGreen > actMinGreen` keeps the arterial from flipping the
instant a single side-street car arrives. `actPassageDistance < actCallDistance`
so a standing queue both *calls* the phase from farther back and *gaps out*
shortly after the last vehicle crosses the stop line. `actMaxGreen` bounds the
minor green so a busy side street can't starve the arterial indefinitely.

## Testing

### Unit tests — `internal/sim/signal_test.go`

- `DefaultSignalConfig`: multi-phase signal → `Plan == PlanSemiActuated`, a valid
  `MajorPhase`, and the major phase is the axis carrying the
  highest-class/highest-speed approach (build a small net where one axis is
  primary and the other residential). Single-phase → `PlanFixed`.
- `AdvanceActuated` — major rest: with no calls, the major phase stays green
  indefinitely (advance many seconds; `PhaseIdx == MajorPhase`, not yellow).
- `AdvanceActuated` — call service: raise a minor call; after `MajorMinGreen` +
  yellow the minor phase goes green, holds ≥ `MinGreen`, and returns to major
  once its call clears and `Passage` elapses.
- `AdvanceActuated` — gap-out: minor green with a vehicle occupying the passage
  zone holds past `MinGreen`; once the zone clears, it terminates after exactly
  `Passage`.
- `AdvanceActuated` — max-out: minor green with the passage zone *continuously*
  occupied terminates at `MaxGreen` (steady demand can't hold green forever).
- `nextActuatedPhase`: skips uncalled minor phases; serves multiple called minor
  phases in cyclic order; falls back to `MajorPhase` when nothing is called.

### Behavior tests — `internal/sim/world_test.go`

- **Arterial rests:** signal with traffic only on the major street and an empty
  side street — the major approach is essentially never stopped (a vehicle
  arriving on the empty side street is what triggers a switch, and there is none).
- **Side street served on demand:** spawn a vehicle on the side street; assert it
  gets a green and clears within `MaxGreen + cycle`, and the major street resumes.
- Existing signal/world behavior tests that assumed fixed 30 s cycling are
  updated to the actuated expectations (or pinned to a `PlanFixed` config where
  they are specifically testing fixed-time mechanics).

### Determinism & suite

- `TestWorld_TraceDeterminism` must still pass unchanged (detection is positional
  and deterministic).
- `go test ./...` green; `go vet ./...` clean.

## Out of scope (deferred)

- Fully-actuated control (major street also detector-actuated / gap-out).
- Coordinated / offset signals along a corridor (green waves).
- A distinct all-red clearance interval (yellow currently covers clearance).
- Actuation parameters in the YAML override schema (overrides stay `PlanFixed`).
- Per-lane / turn-movement detectors and protected-left actuation.
